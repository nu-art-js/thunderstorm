# 2026-05-14 17:34 — Frontend components in Thunderstorm libraries must be themeable via SCSS
- **Context:** While envisioning a file upload UI, the user clarified that all Thunderstorm library frontend components must follow a consistent theming strategy — not just this library, but universally across the framework.
- **Principle:** Every Thunderstorm library that ships frontend components must design them to be styleable via SCSS only. Components ship with a default style (from a Thunderstorm default design language) but use class-based selectors that the consuming app can override with its own dedicated theme. No inline styles or hardcoded visuals — the app-level SCSS theme is the single source of truth for appearance. The library provides structure and default classes; the app provides the look.
